## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, status dot margins |
| sm | 8px | Compact element spacing, dense table cell padding, alert feed item gaps |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, sidebar group spacing |
| xl | 32px | Layout panel gaps, page content padding |
| 2xl | 48px | Major section breaks, auth page vertical spacing |
| 3xl | 64px | Page-level spacing (Welcome page hero section) |

Exceptions:
- Dashboard StatusBar height: 36px (9 * 4, compact monitoring bar)
- Camera marker touch target: 20px (existing, retained for map density)
- Dense table row height: 40px (10 * 4, tighter than default for data grid style per D-12)
- Sidebar collapsed icon-only width: 48px (existing sidebar rail width)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px | 400 (regular) | 1.33 | Status bar text, table metadata, timestamps, similarity scores, sidebar group labels, dense data grid cells |
| Body | 14px | 400 (regular) | 1.5 | Default text, form labels, table cells, alert feed items, nav items |
| Heading | 20px | 600 (semibold) | 1.2 | Page headings, card titles, section headers |
| Display | 28px | 600 (semibold) | 1.2 | Welcome page title, dashboard "Today" stat numbers |

**Monospace:** System monospace (`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace`) at 12px weight 400 for similarity scores, queue depth, device IDs, custom IDs, timestamps in dense data grids.

**Font Weights:** Exactly 2 weights: 400 (regular) for body text and 600 (semibold) for headings and emphasis. Weight 500 (medium) is removed from loading -- existing `font-medium` Tailwind classes are remapped to `font-semibold` where emphasis is needed, or left as `font-normal` where it served as body weight.

## Visual Tiers (per D-02)

### Tier 1: Dashboard (Full Ops Center)
- Darkest backgrounds, maximum data density
- Glassmorphism panels with `backdrop-blur-md`
- Glow effects on severity indicators
- Pulse ring animation on camera markers
- Status bar always visible at bottom
- No whitespace padding around map -- edge-to-edge
- Camera rail and alert feed use compact row height (40px)
- Monospace for all numeric data (similarity, counts, timestamps)
- **Primary visual anchor:** severity-badged alert items in the alert feed -- critical glow draws peripheral attention so operators notice block-list matches without actively scanning

### Tier 2: Admin Pages (Professional Dark)
- Slightly lighter card surfaces (`--card` value)
- Standard whitespace and form spacing (16px gaps)
- Standard table row height (48px) -- still dense but more readable than dashboard
- Severity glow effects retained on badges
- Breadcrumb navigation with standard padding
- Form sections use Card components with subtle border
- More breathing room in Create/Edit forms (24px section gaps)

### Tier 3: Auth Pages (Dark Portal)
- Full dark background matching dashboard palette
- Centered card with glassmorphism effect (per D-06)
- Minimal layout -- logo, form, links only
- Accent color for submit button and link highlights
- Compact form (max-width 400px)

### Tier 4: Settings Pages
- Standard admin tier treatment
- Sidebar sub-navigation with active state indicator
- Form sections with clear separation

## Auth Layout Decision (Claude's Discretion)

**Decision: Switch to AuthCardLayout variant.**

Rationale: The current `AuthSimpleLayout` renders forms directly on the background without a containing card. For the ops portal aesthetic (D-06), a centered card with glassmorphism treatment provides better visual separation against the dark background, creating a more intentional "secure terminal" feel.

**AuthCardLayout specification:**
- Full-viewport dark background (`bg-background`)
- Centered card: `max-w-[400px]`, `rounded-xl`, `border border-border/50`
- Glassmorphism: `backdrop-blur-lg bg-card/75` in dark mode; solid `bg-card` in light mode
- Logo above card (not inside), vertically centered with card
- Card padding: 32px
- Form gap: 16px between fields
- Submit button: full-width, accent color

## Responsive Breakpoints (Claude's Discretion)

| Breakpoint | Value | Behavior |
|------------|-------|----------|
| `sm` | 640px | Stack form columns, hide sidebar (sheet overlay) |
| `md` | 768px | Two-column forms, sidebar visible |
| `lg` | 1024px | Dashboard three-panel layout activates |
| `xl` | 1280px | Maximum content width for admin pages |

Dashboard is designed for `lg+` viewports (command center monitors). Below `lg`:
- Camera rail collapses to hidden (accessible via hamburger menu)
- Alert feed collapses to a bottom sheet
- Map occupies full viewport width

Admin pages below `md`:
- Sidebar becomes sheet overlay (existing behavior, retained)
- Tables become horizontally scrollable with fixed first column
