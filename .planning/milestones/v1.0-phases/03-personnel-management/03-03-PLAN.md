    Create `resources/js/components/PhotoDropzone.vue` implementing D-03, D-04, D-05:

    **Props:**
    ```typescript
    {
        name: string;              // Input name attribute (always "photo")
        currentPhotoUrl?: string;  // Existing photo URL for edit mode (D-04)
        maxSizeBytes?: number;     // Default: 10485760 (10MB upload limit)
        accept?: string;           // Default: "image/jpeg,image/png"
    }
    ```

    **Internal state (refs):**
    - `preview: ref<string | null>(null)` -- data URL from FileReader for preview
    - `error: ref<string | null>(null)` -- inline error message
    - `isDragOver: ref<boolean>(false)` -- whether a drag is over the dropzone
    - `fileInputRef: ref<HTMLInputElement | null>(null)` -- template ref for hidden input
    - `hasNewFile: ref<boolean>(false)` -- whether user has selected a new file

    **Computed:**
    - `displayUrl`: Returns `preview.value` if user selected a new file, else `props.currentPhotoUrl` if in edit mode, else `null`
    - `showReplaceOverlay`: Returns `true` when `displayUrl` is truthy and it's showing the existing photo (not a new selection)

    **Methods:**
    - `onDragEnter(e: DragEvent)`: Set `isDragOver = true`, prevent default
    - `onDragOver(e: DragEvent)`: Prevent default (required for drop to fire)
    - `onDragLeave(e: DragEvent)`: Set `isDragOver = false`
    - `onDrop(e: DragEvent)`: Set `isDragOver = false`, get `e.dataTransfer?.files[0]`, call `handleFile(file)`
    - `onClick()`: Trigger `fileInputRef.value?.click()`
    - `onFileSelect(e: Event)`: Get file from `(e.target as HTMLInputElement).files?.[0]`, call `handleFile(file)`
    - `handleFile(file: File | undefined)`: 
      1. If no file, return
      2. Validate type: if not `file.type.startsWith('image/') && ['image/jpeg', 'image/png'].includes(file.type)` -> set `error = 'Please select a JPEG or PNG image.'`, clear preview, reset file input, return
      3. Validate size: if `file.size > (props.maxSizeBytes ?? 10485760)` -> set `error = 'File is too large. Maximum upload size is 10MB.'`, clear preview, reset file input, return
      4. Clear error
      5. Create FileReader, read as data URL, on load set `preview = reader.result as string`
      6. Set `hasNewFile = true`
      7. Set the hidden file input's files using DataTransfer: `const dt = new DataTransfer(); dt.items.add(file); fileInputRef.value!.files = dt.files;`
    - `removePhoto()`: Clear preview, clear hasNewFile, reset file input value, if in create mode (no currentPhotoUrl) then clear completely

    **Template structure:**
    ```html
    <div class="space-y-1">
        <!-- Dropzone area -->
        <div
            class="relative flex h-48 w-full cursor-pointer items-center justify-center overflow-hidden rounded-lg border-2 border-dashed transition-colors"
            :class="{
                'border-primary bg-accent/50': isDragOver,
                'border-destructive': error,
                'border-border hover:border-muted-foreground/50': !isDragOver && !error,
            }"
            @dragenter.prevent="onDragEnter"
            @dragover.prevent="onDragOver"
            @dragleave="onDragLeave"
            @drop.prevent="onDrop"
            @click="onClick"
        >
            <!-- Preview state: show photo thumbnail -->
            <template v-if="displayUrl">
                <img
                    :src="displayUrl"
                    :alt="'Photo preview'"
                    class="h-full w-full object-cover"
                />
                <!-- Replace overlay for edit mode (D-04) -->
                <div
                    v-if="!hasNewFile && currentPhotoUrl"
                    class="absolute inset-0 flex flex-col items-center justify-center bg-black/50 opacity-0 transition-opacity hover:opacity-100"
                >
                    <Upload class="size-6 text-white" />
                    <span class="mt-1 text-sm text-white">Replace photo</span>
                </div>
                <!-- Remove button for new selections -->
                <button
                    v-if="hasNewFile"
                    type="button"
                    class="absolute right-2 top-2 flex size-6 items-center justify-center rounded-full bg-background/80 hover:bg-background"
                    @click.stop="removePhoto"
                >
                    <X class="size-4" />
                </button>
            </template>

            <!-- Empty state: show upload prompt -->
            <template v-else>
                <div class="flex flex-col items-center gap-2 text-center">
                    <Upload class="size-8 text-muted-foreground/50" />
                    <p class="text-sm text-muted-foreground">
                        Drag and drop a photo here, or click to browse
                    </p>
                </div>
            </template>
        </div>

        <!-- Constraint help text -->
        <p class="text-xs text-muted-foreground">
            JPEG or PNG, max 1MB after processing. Photos are automatically resized.
        </p>

        <!-- Inline client-side error (D-05) -->
        <p v-if="error" class="text-sm text-destructive">
            {{ error }}
        </p>

        <!-- Hidden file input -->
        <input
            ref="fileInputRef"
            type="file"
            :name="name"
            :accept="accept ?? 'image/jpeg,image/png'"
            class="hidden"
            @change="onFileSelect"
        />
    </div>
    ```

    **Imports needed:**
    ```typescript
    import { Upload, X } from 'lucide-vue-next';
    import { computed, ref } from 'vue';
    ```

    **Important implementation details:**
    - The hidden `<input type="file" :name="name">` must have `name="photo"` to match the FormRequest field name. Inertia's `<Form>` component collects data from native form elements by their `name` attribute.
    - Using `DataTransfer` API to programmatically set files on the hidden input ensures the file is included in form submission.
    - The `@click.stop` on the remove button prevents the click from bubbling to the dropzone and re-opening the file picker.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user browser -> dropzone | Client-side file validation (type, size) before upload |
| dropzone -> Form -> server | File sent via multipart/form-data through Inertia |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | PhotoDropzone | mitigate | Client-side validation checks file.type against ['image/jpeg', 'image/png'] and file.size against maxSizeBytes. Server-side FormRequest independently validates 'image', 'mimes:jpeg,png', 'max:10240'. Client validation is UX only -- server is the security boundary. |
| T-03-10 | Tampering | Create/Edit forms | mitigate | Inertia `<Form>` component automatically includes CSRF token. Wayfinder `.form()` generates correct method and URL. No manual CSRF handling needed. |
| T-03-11 | Denial of Service | PhotoDropzone | accept | Client-side 10MB limit prevents unnecessarily large uploads from reaching the server. Server-side 'max:10240' rule provides authoritative limit. PHP upload_max_filesize is the ultimate backstop. |
</threat_model>
