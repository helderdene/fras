## Critical Integration Gaps

### 1. CameraStatusChanged Missing `broadcastAs()` (CRITICAL)

**Root cause:** `CameraStatusChanged` event (Phase 2) does not define `broadcastAs()`. Laravel broadcasts it under the full PHP class name `App\Events\CameraStatusChanged`. Vue Echo listeners register for `.CameraStatusChanged` (short name) — these never match.

**Contrast:** `RecognitionAlert` and `EnrollmentStatusChanged` both correctly define `broadcastAs()` and work.

**Fix:** Add `public function broadcastAs(): string { return 'CameraStatusChanged'; }` to `app/Events/CameraStatusChanged.php`.

**Affected requirements:** CAM-03, CAM-05, DASH-02, DASH-05, OPS-04

**Broken flows:**
- Camera online/offline → real-time marker color change on Dashboard
- Camera online/offline → real-time status dot on cameras/Index and cameras/Show
- MQTT activity heuristic for StatusBar MQTT indicator

### 2. Echo Client .env.example Mismatch (CRITICAL)

**Root cause:** Project switched from Reverb to Pusher cloud. `app.ts` uses `VITE_PUSHER_APP_KEY` and `VITE_PUSHER_APP_CLUSTER`, but `.env.example` only contains `VITE_REVERB_*` variables. `global.d.ts` TypeScript declarations only declare `VITE_REVERB_*` types.

**Fix:** Update `.env.example` to include `VITE_PUSHER_APP_KEY` and `VITE_PUSHER_APP_CLUSTER`. Update `global.d.ts` to declare matching types. Remove or mark obsolete the `VITE_REVERB_*` entries.

**Affected requirements:** INFRA-03 (new developer WebSocket setup fails silently)
