trait ProfileValidationRules
{
    protected function profileRules(?int $userId = null): array
    {
        return [
            'name' => $this->nameRules(),
            'email' => $this->emailRules($userId),
        ];
    }

    protected function nameRules(): array
    {
        return ['required', 'string', 'max:255'];
    }

    protected function emailRules(?int $userId = null): array
    {
        return [
            'required', 'string', 'email', 'max:255',
            $userId === null
                ? Rule::unique(User::class)
                : Rule::unique(User::class)->ignore($userId),
        ];
    }
}

**1. Disable registration (per D-01):**

In `config/fortify.php`, comment out `Features::registration()` on line 147. Change it to:
```php
// Features::registration(),
```
Do NOT delete Register.vue or CreateNewUser action (per D-02 -- keep as dead code).

**2. Create StoreUserRequest** at `app/Http/Requests/User/StoreUserRequest.php`:
- Use `php artisan make:request User/StoreUserRequest --no-interaction`
- `authorize()` returns `true`
- Use `PasswordValidationRules` and `ProfileValidationRules` traits
- `rules()` returns: `...profileRules()` spread + `'password' => passwordRules()`
- This gives: name (required|string|max:255), email (required|string|email|max:255|unique:users), password (required|string|Password::default()|confirmed)

**3. Create UpdateUserRequest** at `app/Http/Requests/User/UpdateUserRequest.php`:
- Use `php artisan make:request User/UpdateUserRequest --no-interaction`
- `authorize()` returns `true`
- Use `ProfileValidationRules` trait
- `rules()` returns: `...profileRules($this->route('user')->id)` spread + `'password' => ['nullable', 'string', Password::default(), 'confirmed']`
- The password field is nullable on update -- when provided, sets new password; when omitted, password unchanged (per D-06)

**4. Create UserController** at `app/Http/Controllers/UserController.php`:
- Use `php artisan make:controller UserController --resource --no-interaction`
- Follow CameraController pattern exactly. Import: StoreUserRequest, UpdateUserRequest, User model, Inertia, Response, RedirectResponse

Methods:
- `index()`: Return `Inertia::render('users/Index', ['users' => User::orderBy('name')->get(['id', 'name', 'email', 'created_at'])])`
- `create()`: Return `Inertia::render('users/Create')`
- `store(StoreUserRequest $request)`: `User::create($request->validated())`, flash toast 'User created.', redirect to `users.index`
- `edit(User $user)`: Return `Inertia::render('users/Edit', ['user' => $user])`
- `update(UpdateUserRequest $request, User $user)`: Filter out null password with `$data = $request->validated(); if (empty($data['password'])) { unset($data['password']); }`, then `$user->update($data)`, flash toast 'User updated.', redirect to `users.index`
- `destroy(User $user)`: Prevent self-deletion with `if ($user->id === $request->user()->id) { return back()->withErrors(['user' => 'You cannot delete your own account.']); }`, then `$user->delete()`, flash toast 'User deleted.', redirect to `users.index`
- No `show()` method needed (edit serves as detail view). Override resource to exclude show.

**5. Add routes** in `routes/web.php`:
- Add `use App\Http\Controllers\UserController;` import at top
- Inside the `Route::middleware(['auth', 'verified'])->group(...)` closure, add:
  `Route::resource('users', UserController::class)->except(['show']);`
- Place it after the `Route::resource('personnel', PersonnelController::class);` line
