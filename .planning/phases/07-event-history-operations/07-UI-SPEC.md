## Layout Specification

### Page Structure: events/Index.vue

```
+---------------------------------------------------------------+
| Heading: "Event History"                                       |
+---------------------------------------------------------------+
| Filter Bar (EventHistoryFilters)                               |
| [Date From] [Date To] [Camera v] [Search person...] [All|Cri|War|Info] |
+---------------------------------------------------------------+
| Data Table (EventHistoryTable)                                 |
| Face | Person      | Camera    | Severity | Similarity | Time  |
| [av] | John Doe    | Gate A    | [Crit]   | 92.3%     | 14:30 |
| [av] | Jane Smith  | Gate B    | [Info]   | 88.1%     | 14:28 |
| ...                                                            |
+---------------------------------------------------------------+
| Pagination (EventHistoryPagination)                            |
| < 1 2 3 ... 12 >        Showing 1-25 of 293 events           |
+---------------------------------------------------------------+
```

### Filter Bar Layout

- **Direction:** Horizontal row, `flex items-center gap-2 flex-wrap`
- **Date From input:** type="date", width 160px, default value = today's date
- **Date To input:** type="date", width 160px, default value = today's date
- **Camera Select:** shadcn Select, width 180px, placeholder "All cameras", options from Inertia prop
- **Person Search:** shadcn Input, width 200px, placeholder "Search person...", debounced 300ms
- **Severity Pills:** Button group matching alerts/Index.vue pattern -- All (default active) | Critical | Warning | Info
- **Responsive:** On mobile (< 768px), filter bar wraps to 2 rows. Date inputs on first row, camera + search on second row, pills on third row.

### Table Column Specification

| Column | Header Label | Width | Sortable | Content |
|--------|-------------|-------|----------|---------|
| Face | (no label, icon header) | 48px fixed | No | Avatar 32x32 with face crop, AvatarFallback with initials |
| Person | Person | flex-1 min-w-[140px] | Yes (sort by personnel name) | Person name (14px/400), custom ID below in muted-foreground (12px/400) |
| Camera | Camera | 140px | Yes (sort by camera name) | Camera name (14px/400) |
| Severity | Severity | 100px | Yes (sort by severity) | SeverityBadge component + optional "Replay" badge |
| Similarity | Similarity | 90px | Yes (sort by similarity desc) | Monospace 12px, formatted as XX.X% |
| Timestamp | Time | 160px | Yes (sort by captured_at, default desc) | Absolute datetime: "Apr 11, 2026, 2:30 PM" via Intl.DateTimeFormat |

**Default sort:** captured_at descending (newest first).

**Sort interaction:** Clicking a sortable column header triggers an Inertia visit with `sort` and `direction` query params. Active sort column header shows a small arrow icon (ChevronUp/ChevronDown from lucide) in accent color.

### Table Row Behavior

- **Click:** Opens AlertDetailModal with the row's RecognitionEvent data. Entire row is clickable (`cursor-pointer`).
- **Hover:** `hover:bg-muted/50` background tint.
- **Replay events:** Row has `opacity-60` class. A small `bg-muted text-muted-foreground` badge reading "Replay" appears inline after the SeverityBadge.
- **Missing face image:** AvatarFallback with person initials (matches AlertFeedItem pattern).
- **Missing person (null personnel):** Display `name_from_camera` field, or "Unknown" if also null.
- **Acknowledged events:** No special visual treatment in history table (ack status visible in modal).
- **Dismissed events:** No special visual treatment in history table (dismiss status visible in modal).

### Pagination

- **Style:** Numbered page pagination via shadcn Pagination component.
- **Per page:** 25 rows.
- **URL params:** `?page=N` appended to current filter URL. Bookmarkable.
- **Display:** "Showing X-Y of Z events" text in muted-foreground (12px) alongside pagination controls.
- **Behavior:** Each page button triggers Inertia visit preserving all current filter params.

### Empty State

- **Container:** Card with 48px vertical padding, centered content.
- **Icon:** `Search` from lucide-vue-next, 64px, `text-muted-foreground/50`.
- **When no events exist at all (fresh system):**
  - Heading: "No recognition events yet"
  - Body: "Events will appear here as cameras detect personnel. Check that cameras are online and enrolled."
- **When filters return no results:**
  - Heading: "No matching events"
  - Body: "Try adjusting your filters or expanding the date range."

## Interaction Contract

### Filter State Management

- All filter state is expressed as URL query parameters: `?date_from=2026-04-11&date_to=2026-04-11&camera_id=3&search=john&severity=critical&sort=captured_at&direction=desc&page=1`
- Changing any filter resets page to 1.
- Filters trigger Inertia `router.get()` visits with `preserveState: true` and `replace: true`.
- Person search input is debounced at 300ms before triggering the visit.
- Date inputs use native HTML `<input type="date">` for simplicity and browser-native date picking.
- Camera select populated from `cameras` Inertia prop (eager-loaded, small dataset).

### Server-Side Query Parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `date_from` | string (Y-m-d) | today | Start date inclusive |
| `date_to` | string (Y-m-d) | today | End date inclusive |
| `camera_id` | integer or null | null (all) | Filter by camera |
| `search` | string or null | null | Person name or custom ID partial match |
| `severity` | string or null | null (all) | critical, warning, or info |
| `sort` | string | captured_at | Column to sort by |
| `direction` | string | desc | asc or desc |
| `page` | integer | 1 | Current page |

### Keyboard Accessibility

- Tab order: filter controls left-to-right, then table rows top-to-bottom, then pagination.
- Table rows are focusable (`tabindex="0"`) and activate on Enter key (opens modal).
- Sort column headers are `<button>` elements within `<th>`.
- Filter pill buttons use standard Button component (already accessible).
