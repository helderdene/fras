## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> EventHistoryController | User-controlled query parameters (sort, search, date_from, date_to, camera_id, severity, page) cross into Eloquent queries |
| CleanupRetentionImagesCommand -> Storage | Command deletes files from disk based on database path values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-01 | Tampering | EventHistoryController sort param | mitigate | Whitelist sort columns: `in_array($request->input('sort'), ['captured_at', 'similarity', 'severity'], true)` with fallback to 'captured_at'. Never interpolate raw user input into ORDER BY. |
| T-7-02 | Tampering | EventHistoryController search param | mitigate | Eloquent `where('col', 'like', "%{$search}%")` uses prepared statements with automatic parameter binding. No raw SQL. |
| T-7-03 | Information Disclosure | EventHistoryController unauthorized access | mitigate | Route registered inside `['auth', 'verified']` middleware group, same as AlertController and all other protected routes. |
| T-7-04 | Tampering | Image serving path traversal | accept | No new image serving routes created. Existing AlertController::faceImage/sceneImage routes use Eloquent model binding + Storage::disk pattern. No user-controlled path input. |
| T-7-05 | Denial of Service | CleanupRetentionImagesCommand large batch | mitigate | Uses `chunkById(200)` for memory-efficient iteration. Schedule entry uses `withoutOverlapping()` to prevent concurrent runs. |
| T-7-06 | Tampering | EventHistoryController direction param | mitigate | Validate `$request->input('direction') === 'asc' ? 'asc' : 'desc'` -- binary choice, no user string in query. |
</threat_model>
