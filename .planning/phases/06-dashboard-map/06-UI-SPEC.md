## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status dot margins |
| sm | 8px | Compact element spacing, left rail row padding, status bar indicator gaps |
| md | 16px | Default element spacing, panel internal padding, map popup padding |
| lg | 24px | Section padding within panels, stats grid gap |
| xl | 32px | Not used in this phase (full-viewport layout has no large section breaks) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Left rail width fixed at 280px (D-01). Right alert feed width fixed at 360px (D-01). Status bar height fixed at 36px. Top navigation bar height fixed at 48px. These are layout dimensions, not spacing tokens.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.33 |
| Heading | 16px | 600 (semibold) | 1.25 |
| Stat Value | 24px | 600 (semibold) | 1.2 |

Notes:
- Body at 14px (not 16px) for information density -- this is a command center, not a content page. Matches existing `text-sm` usage in AlertFeedItem and camera table rows.
- Label at 12px for status bar indicators, stat labels, camera recognition count badges. Maps to `text-xs`.
- Heading at 16px for panel headers ("Cameras", "Live Alerts", stat panel title). Maps to `text-base font-semibold`.
- Stat Value at 24px for the four "Today" statistics numbers in the 2x2 grid. Maps to `text-2xl font-semibold`.

### Pulse Ring Animation Color

The expanding ring animation uses `rgba(239, 68, 68, 0.6)` (red-500 at 60% opacity) fading to `rgba(239, 68, 68, 0)` over 3 seconds. This is a CSS keyframe animation on a Mapbox GL JS marker HTML element, not a Tailwind utility.

## Layout Contract

### Three-Panel Full-Viewport Layout (D-01, D-02, D-04)

```
+--[Top Nav Bar 48px]----------------------------------------------+
|  Logo   |                                    | Theme Toggle | Nav |
+--[Left Rail 280px]--+--[Map (flex-1)]--+--[Alert Feed 360px]-----+
|  "Today" Stats      |                  |  Filter Pills            |
|  2x2 grid           |   Mapbox GL JS   |  AlertFeedItem           |
|  ----------------   |   Full map       |  AlertFeedItem           |
|  Camera List        |   Camera markers |  AlertFeedItem           |
|  - Cam 1  [3]       |   Pulse rings    |  ...                     |
|  - Cam 2  [0]       |                  |                          |
|  - Cam 3  [1]       |                  |                          |
+---------------------+------------------+--------------------------+
|  [Status Bar 36px] MQTT: Connected  Reverb: Connected  Queue: 0  |
+------------------------------------------------------------------+
```

- Full viewport: `h-screen w-screen overflow-hidden`
- Top nav: `h-12` (48px), `bg-muted`, border-bottom
- Left rail: `w-[280px]`, `bg-muted/50`, border-right, vertical scroll for camera list
- Center map: `flex-1`, fills all remaining space
- Right alert feed: `w-[360px]`, `bg-background`, border-left, vertical scroll
- Status bar: `h-9` (36px), `bg-muted`, border-top
- Panels togglable open/closed (D-01) but NOT drag-resizable

### Left Rail (D-11, D-12, D-13)

**"Today" Statistics Panel (D-12):**
- 2x2 grid at top of left rail
- Each cell: stat value (24px semibold) over label (12px medium muted-foreground)
- Grid gap: 8px
- Cells:
  1. Total recognitions today (number)
  2. Critical events today (number, red-500 text)
  3. Warnings today (number, amber-500 text)
  4. Enrolled personnel (number)
- Container: `p-4`, `border-b border-border`

**Camera List (D-11, D-13):**
- Each row: `px-4 py-2`, `cursor-pointer`, `hover:bg-muted`
- Row contents: status dot (6px circle, emerald-500 or neutral-400) + camera name (14px truncated) + recognition count badge (12px, `bg-muted rounded-full px-2 py-0.5`)
- Active camera (selected): `bg-accent text-accent-foreground`
- Click action: pan map to camera marker, open popup, filter right alert feed to that camera (D-13)
- Click again or click "All Cameras" header: clear filter, close popup
- Scroll: `overflow-y-auto` when list exceeds viewport

### Right Alert Feed (D-04, D-13)

- Reuses existing `AlertFeedItem.vue` component directly
- Reuses existing `AlertDetailModal.vue` for click-to-detail
- Filter pills at top: "All", "Critical", "Warning", "Info" (same pattern as alerts/Index.vue)
- Camera filter integration: when a camera is selected in the left rail, feed shows only that camera's events. A chip or label shows "Filtered: {camera name}" with an X to clear.
- Real-time updates via Echo on `fras.alerts` channel (same pattern as alerts/Index.vue)
- Cap at 50 items in the reactive array (D-03 from Phase 5 context)
- Sound composable (`useAlertSound`) active for critical events

### Status Bar (D-09, D-10)

| Indicator | Visual | Values |
|-----------|--------|--------|
| MQTT | 6px circle + "MQTT" label | Green = connected, Red = disconnected |
| Reverb | 6px circle + "Reverb" label | Green = connected, Red = disconnected |
| Queue | Number + "pending" label | Shows count of pending queue jobs |

- Layout: `flex items-center gap-6 px-4`
- All text at 12px medium weight
- Dots: `size-1.5 rounded-full`

### Connection Loss Banner (D-09)

- Appears below top nav bar when MQTT or Reverb is disconnected
- Background: `bg-amber-500/10 border-b border-amber-500/20`
- Text: "Real-time connection lost. Alerts may be delayed." at 12px, `text-amber-700 dark:text-amber-400`
- Auto-dismisses when connection is restored
- No manual dismiss button

## Camera Markers Contract (D-05, D-06, D-07)

### Marker Appearance

- Custom HTML markers (not default Mapbox markers) for styling control
- Online: 12px circle, `bg-emerald-500`, `border-2 border-white dark:border-neutral-900`, `shadow-sm`
- Offline: 12px circle, `bg-neutral-400 dark:bg-neutral-500`, `border-2 border-white dark:border-neutral-900`
- Positioned by camera GPS coordinates (latitude, longitude from Camera model)

### Marker Popup (D-05)

- Opens on marker click
- Content: camera name (14px semibold), status dot + "Online"/"Offline" (12px), last seen relative time (12px muted), recognition count today (12px), link "View Details" to camera show page
- Styling: `bg-popover text-popover-foreground`, `rounded-lg`, `shadow-lg`, `p-4`, max-width 220px
- Close on click outside or clicking another marker

### Pulse Ring Animation (D-06)

- Trigger: when a `RecognitionAlert` broadcast fires for a camera
- Visual: a circle expanding from 12px to 48px centered on the camera marker, starting at `rgba(239, 68, 68, 0.6)` and fading to `rgba(239, 68, 68, 0)` over 3 seconds
- Implementation: CSS keyframes on an absolutely-positioned div appended to the marker's HTML container. Multiple overlapping rings allowed (D-06).
- Keyframes:

```css
@keyframes pulse-ring {
  0% {
    width: 12px;
    height: 12px;
    opacity: 0.6;
  }
  100% {
    width: 48px;
    height: 48px;
    opacity: 0;
  }
}
```

- Duration: 3s ease-out, no repeat. Element removed from DOM after animation completes.

## Map Behavior Contract (D-06, D-08)

### Style Toggle (D-08)

- Dark style URL: `config('hds.mapbox.dark_style')` -- custom HelderDene Mapbox Studio style
- Light style URL: `config('hds.mapbox.light_style')` -- custom HelderDene Mapbox Studio style
- Toggle switches BOTH the Mapbox map style AND the app dark/light theme together via `useAppearance` composable
- On toggle: call `map.setStyle(newStyleUrl)` then re-add all markers (Mapbox removes custom layers on style change)

### Initial View

- Center: Butuan City coordinates (8.9475, 125.5406) -- same defaults as existing MapboxMap.vue
- Zoom: 15 (existing default)
- If cameras have GPS coordinates, fit bounds to show all markers on load with 48px padding

### Interaction

- Interactive: true (pan, zoom, rotate enabled)
- Scroll zoom: enabled
- Touch zoom/rotate: enabled
- Attribution: hidden (`attributionControl: false`, matching existing MapboxMap pattern)
- Map resizes when panels are toggled via `map.resize()` after panel transition completes

## Interaction States

### Camera Selection Flow (D-11, D-13)

1. **No camera selected (default):** All cameras shown in left rail without highlight. Alert feed shows all events. "All Cameras" header is not visually active.
2. **Camera clicked in left rail:** Row highlights with `bg-accent text-accent-foreground`. Map pans to camera, popup opens. Alert feed filters to that camera. "Showing: {name}" chip appears above feed.
3. **Same camera clicked again:** Selection cleared. Row de-highlights. Map stays at current position. Alert feed shows all events. Chip removed.
4. **Different camera clicked:** Previous selection cleared, new selection applied. Map pans to new camera.
5. **Camera marker clicked on map:** Same as clicking in left rail -- left rail highlights, feed filters.
6. **"All Cameras" clicked:** Selection cleared (same as clicking selected camera again).

### Panel Toggle Flow (D-01)

1. **Both panels open (default):** Left rail 280px, right feed 360px, map fills center.
2. **Left rail hidden:** Map expands left. `map.resize()` called after 200ms transition.
3. **Right feed hidden:** Map expands right. `map.resize()` called after 200ms transition.
4. **Both hidden:** Map fills full width between top nav and status bar.
5. **Toggle animation:** CSS transition `width 200ms ease-in-out` on panels. Panels slide out; map fills the space.

### Real-Time Update Flow

1. **RecognitionAlert received:** Alert prepended to feed, pulse ring animates on camera marker, if critical then sound plays, stats counts update.
2. **CameraStatusChanged received:** Camera marker color updates, left rail status dot updates, popup content updates if open.
3. **Connection lost:** Status bar dot turns red, connection banner slides in below top nav.
4. **Connection restored:** Status bar dot turns green, connection banner slides out.

## Accessibility

- Map container: `role="application"` `aria-label="Camera locations map"` (matching existing MapboxMap)
- Left rail camera list: `role="listbox"` `aria-label="Camera list"`
- Camera rail items: `role="option"` `aria-selected="true/false"`
- Alert feed: `aria-live="polite"` region for new alert announcements (matching existing alerts/Index.vue pattern)
- Panel toggle buttons: `aria-expanded="true/false"` `aria-controls="{panel-id}"`
- Status indicators: `aria-label="MQTT: connected"` / `aria-label="MQTT: disconnected"`
- Theme toggle: `aria-label="Switch to dark mode"` / `aria-label="Switch to light mode"`
- Keyboard: Tab order flows top nav -> left rail -> map -> right feed. Escape closes map popup.
