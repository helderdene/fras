<objective>
Display the acknowledging operator's name on acknowledged alerts in the alert feed, alert detail modal, and event history, fulfilling REC-13 ("Operator can acknowledge or dismiss an alert, recording who handled it and when").

Purpose: Currently acknowledged alerts show only "Acknowledged at {timestamp}" but not who acknowledged them. For a command center with multiple operators, knowing WHO handled an alert is critical for accountability. The backend acknowledge endpoint and model already track the user ID -- this plan wires the name through to the frontend.

Output: Operator name visible on all acknowledged alerts across alert feed, detail modal, and event history table.
</objective>
