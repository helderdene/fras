| Token | Value | Usage |

| sm | 8px | Compact alert row internal padding, badge padding |
| md | 16px | Alert feed row padding, modal section gaps |

| Role | Size | Weight | Line Height |

Phase-specific notes:
- Alert feed rows use 14px body for person name and camera, 12px label for similarity score and timestamp
- Severity badge text uses 12px (text-xs) with medium weight per existing Badge variant (font-medium)
- Modal metadata labels use 12px (text-xs) muted-foreground, values use 14px (text-sm)
- "No alerts yet" empty state heading uses 18px (text-lg) semibold

Source: Matches existing codebase patterns (cameras/Index.vue uses text-sm body, text-xs for device IDs, text-lg for empty state headings)

## Interaction Contracts

### Alert Feed (alerts/Index.vue)

**Layout:**
- Full-width content area within AppLayout (no split pane in this phase; dashboard layout is Phase 6)
- Filter pills row at top: `All | Critical | Warning | Info` (D-04)
- Alert sound toggle button in top-right of header area (D-09)
- Scrollable feed below, reverse-chronological (D-01)

**Feed Row (AlertFeedItem.vue):**
- Height: compact, approximately 56-64px (py-3, single line of metadata)
- Left border: 4px solid, colored by severity (D-01, REC-09)
- Background: subtle severity tint on the row (D-01)
- Content left-to-right: face crop Avatar (32px, size-8), person name + camera name (stacked or inline), severity Badge, similarity score (text-xs mono), relative timestamp (text-xs muted)
- Hover state: row background darkens slightly (`hover:bg-muted/50`), Ack/Dismiss icon buttons fade in on the right side (D-10)
- Click: opens AlertDetailModal for that event (D-05)

**New Alert Animation (D-02):**
- New alerts prepend to the list, pushing existing alerts down
- New alert row has a brief highlight flash animation (200-300ms `bg-primary/10` fade to normal severity tint) using `tw-animate-css` or CSS transition
- Feed capped at 50 items; excess items are silently removed from the bottom (D-03)

**Filter Pills (D-04):**
- Row of toggle buttons: All (default active), Critical, Warning, Info
- Use Button component with `variant="outline"` for inactive, `variant="default"` for active
- Filtering is client-side on the reactive alerts array
- Show count next to each label: "Critical (3)"

**Alert Sound Toggle (D-09):**
- Icon button with Bell icon (lucide `Bell`) when disabled, BellRing when enabled
- Click triggers user gesture for audio unlock
- Visual indicator: muted-foreground when off, primary when enabled
- Tooltip: "Enable alert sounds" / "Mute alert sounds"

**Acknowledge/Dismiss (D-10):**
- Inline icon buttons visible on hover/focus of alert row
- Acknowledge: Check icon (lucide `Check`), sends POST to acknowledge endpoint via `useHttp`
- Dismiss: X icon (lucide `X`), sends POST to dismiss endpoint via `useHttp`
- After Ack: row shows "Acknowledged by [user] at [time]" in muted text below metadata
- After Dismiss: row fades to 50% opacity, remains in feed

**Real-Time Updates:**
- Echo listener on `fras.alerts` channel for `.RecognitionAlert` events
- Payload contains all data needed to render a feed row (no additional API calls)
- Critical events trigger `playAlertSound()` from `useAlertSound` composable (D-08)

### Alert Detail Modal (AlertDetailModal.vue)

**Trigger:** Click on any AlertFeedItem row

**Layout (D-05):**
- Wide modal: `max-w-2xl` (672px) using DialogContent
- Top section: side-by-side images
  - Left: Face crop, 150px width, rounded corners, Avatar component at larger size
  - Right: Scene image (SceneImageOverlay), fills remaining width, with yellow bounding box overlay from target_bbox
  - If scene image missing (D-07): gray placeholder box (`bg-muted`) with centered text "Scene image not available" in `text-muted-foreground text-sm`
- Middle section: Metadata grid (2 columns)
  - Person name, Custom ID, Camera name, Similarity score (formatted as percentage), Person type (Badge: Allow/Block), Captured timestamp (absolute format), Severity (SeverityBadge)
- Footer section: Action buttons
  - "Acknowledge" button (primary variant) -- records current user + timestamp
  - "Dismiss" button (outline variant) -- marks as dismissed
  - If already acknowledged: show "Acknowledged by [name] at [time]" text instead of Acknowledge button

### Empty State

**When:** No recognition events exist yet (feed has zero items)

**Layout:**
- Centered within Card, matching cameras/Index.vue and personnel/Index.vue empty state pattern
- Icon: `ShieldAlert` from lucide (64px, muted-foreground/50)
- Heading: "No alerts yet" (text-lg font-semibold)
- Body: "Recognition alerts will appear here in real time as cameras detect known personnel." (text-sm text-muted-foreground)
- No CTA button (alerts come from cameras, not user action)

### Error States

**WebSocket disconnected:**
- Subtle banner at top of feed: "Real-time connection lost. Alerts may be delayed." with amber background (`bg-amber-50 dark:bg-amber-950/30`)
- Auto-dismiss when reconnected

**Image load failure:**
- Face crop: show AvatarFallback with person initials (existing pattern)
- Scene image in modal: show gray placeholder with "Failed to load scene image"

## Animation Contract

| Animation | Trigger | Duration | Implementation |
|-----------|---------|----------|----------------|
| New alert highlight | Alert prepended to feed | 300ms | `transition-colors duration-300` from `bg-primary/10` to normal severity tint |
| Alert slide-in | Alert prepended to feed | 200ms | CSS `transition: max-height 200ms ease-out` or Vue `<TransitionGroup>` with `move` class |
| Dismiss fade | Dismiss button clicked | 200ms | `transition-opacity duration-200` to `opacity-50` |
| Ack/Dismiss buttons appear | Row hover or focus-within | 150ms | `opacity-0 group-hover:opacity-100 transition-opacity duration-150` |
| Modal open/close | Click feed row / close modal | Default shadcn | Built into Dialog component animations |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Feed row keyboard navigation | Rows are focusable (`tabindex="0"`), Enter opens detail modal |
| Ack/Dismiss keyboard access | Icon buttons receive focus on Tab within row, Enter/Space triggers action |
| Severity announcement | `aria-label` on severity badge: "Severity: Critical" |
| Alert sound status | `aria-live="polite"` region announces "Alert sounds enabled" / "Alert sounds muted" |
| New alert announcement | `aria-live="polite"` region announces "New critical alert from [camera]" for critical events |
| Modal focus trap | Built into shadcn Dialog (reka-ui DialogContent traps focus) |
| Color not sole indicator | Severity communicated via text label (badge), border color, AND background tint |
| Relative time aria | Timestamp includes `title` attribute with absolute datetime for screen readers |
