## Page Specifications

### Personnel Index (`pages/personnel/Index.vue`)

**Layout:** Full-width within AppLayout content area
**Pattern source:** `pages/cameras/Index.vue`

**Structure:**
1. Header row: `Heading` (title="Personnel") on left, `Button` (Add Personnel) with Plus icon on right -- `flex items-center justify-between`
2. Search bar: `Input` with search placeholder above table -- `max-w-sm` width
3. Table: `<table class="w-full text-sm">` with `overflow-x-auto` wrapper
4. Empty state: `Card` with `CardContent`, Users icon (size-12), heading, body text, CTA button

**Table columns:**

| Column | Width | Content | Alignment |
|--------|-------|---------|-----------|
| Avatar | 48px fixed | Avatar (size-8 / 32px) with AvatarImage + AvatarFallback (initials) | Left |
| Name | flex-1 | Link to show page, `text-foreground hover:underline` | Left |
| Custom ID | 120px | `font-mono text-xs text-muted-foreground` | Left |
| Type | 80px | Badge (secondary for Allow, destructive for Block) | Left |
| Sync Status | 100px | SyncStatusDot (gray "Not synced" placeholder) | Left |

**Table row:** `border-b border-border hover:bg-muted/50` (matches camera pattern)

**Search behavior (D-07):** Client-side filter-as-you-type. `computed` filters `personnel` array by `name` or `custom_id` containing search string (case-insensitive). No debounce needed for ~200 records.

### Personnel Show (`pages/personnel/Show.vue`)

**Layout:** 3/5 + 2/5 grid at lg breakpoint -- `grid grid-cols-1 gap-6 lg:grid-cols-5`
**Pattern source:** `pages/cameras/Show.vue`

**Left panel (lg:col-span-3):** Card with personnel information
1. Large photo: Avatar component at 200px (`size-[200px]`) centered in card with rounded-lg overflow-hidden. AvatarFallback shows initials at text-4xl.
2. Info fields (read-only, stacked): Each field as `<div>` with `<p class="text-sm text-muted-foreground">Label</p>` + `<p class="text-sm text-foreground">Value</p>` -- matches camera Show pattern exactly
3. Field groups with Separator between sections:
   - **Identity:** Name, Custom ID, Person Type (Badge)
   - **Details:** Gender, Birthday, ID Card
   - **Contact:** Phone, Address
4. Action buttons: `flex items-center gap-3` with Edit (variant="outline") and Delete (variant="destructive") -- mirrors camera Show

**Right panel (lg:col-span-2):** Card with enrollment sidebar (D-10)
- CardHeader: CardTitle "Enrollment Status"
- CardContent: List of all cameras, each row showing camera name + SyncStatusDot (gray "Not synced")
- If no cameras registered: empty state with CameraIcon, "No cameras registered" heading, "Register cameras to enable enrollment" body text

**Delete dialog:** Matches camera pattern exactly
- DialogTitle: "Delete personnel?"
- DialogDescription: "This will permanently remove {name} and all associated enrollment records. This action cannot be undone."
- DialogFooter: "Keep Personnel" (secondary) + "Delete Personnel" (destructive)

### Personnel Create (`pages/personnel/Create.vue`)

## PhotoDropzone Component Specification

**File:** `resources/js/components/PhotoDropzone.vue`

**Props:**
```typescript
{
    name: string;              // Input name attribute (always "photo")
    currentPhotoUrl?: string;  // Existing photo URL for edit mode (D-04)
    maxSizeBytes?: number;     // Default: 10485760 (10MB upload limit)
    accept?: string;           // Default: "image/jpeg,image/png"
}
```

**States:**

| State | Visual |
|-------|--------|
| Empty (no photo) | Dashed border (border-2 border-dashed border-border), Upload icon (size-8 text-muted-foreground/50), "Drag and drop a photo here, or click to browse" text, constraint help text below |
| Drag over | Dashed border changes to border-primary, background changes to bg-accent/50 |
| Preview (photo selected) | Photo thumbnail fills the dropzone area (object-cover rounded-lg), remove button (X icon, absolute top-right, size-6 bg-background/80 rounded-full) |
| Existing photo (edit mode, D-04) | Photo thumbnail displayed, "Replace" overlay on hover (semi-transparent bg-black/50 with Upload icon and "Replace photo" text) |
| Client-side error (D-05) | Dashed border changes to border-destructive, error message displayed inline below the dropzone (text-sm text-destructive) |

**Dimensions:** `h-48 w-full` (192px height, full width). Rounded corners `rounded-lg`.

**Constraint help text:** `<p class="mt-2 text-xs text-muted-foreground">JPEG or PNG, max 1MB after processing. Photos are automatically resized.</p>`

**Client-side validation (D-05):**
- Reject non-image files: Show "Please select a JPEG or PNG image"
- Reject files over 10MB: Show "File is too large. Maximum upload size is 10MB."
- Valid file: Show preview thumbnail, clear any error

**Implementation approach:** Vanilla browser APIs (dragenter/dragover/drop/dragleave events) with hidden `<input type="file" name="photo" accept="image/jpeg,image/png">`. FileReader API for client-side preview. Optionally use `@vueuse/core` `useDropZone` composable.

## Interaction Patterns

### Form Submission
- `<Form v-bind="Controller.action.form()">` pattern (Wayfinder)
- Submit button shows disabled state during processing (`:disabled="processing"`)
- Server validation errors display via `<InputError :message="errors.field">` below each field
- Success redirects with toast notification via `Inertia::flash('toast', ...)`

### Navigation
- Breadcrumbs: `Personnel > {page}` or `Personnel > {name} > Edit`
- Sidebar: "Personnel" nav item with Users icon (added to `AppSidebar.vue`)
- All navigation via Wayfinder route functions (never hardcoded URLs)

### Delete Flow
- Trigger: "Delete" button (variant="destructive") on Show page
- Dialog opens with confirmation copy
- Cancel: "Keep Personnel" (variant="secondary") closes dialog
- Confirm: Form submits via `PersonnelController.destroy.form(personnel)`
- Success: Redirect to personnel index with "Personnel deleted." toast

### Photo Upload Flow
1. User drags photo onto dropzone OR clicks to open file picker
2. Client validates file type and size immediately (D-05)
3. If invalid: show inline error, do not set file
4. If valid: show thumbnail preview via FileReader
5. On form submit: file sent as multipart/form-data (Inertia `<Form>` handles this automatically)
6. Server preprocesses: orient, resize to 1080p, compress to JPEG, compute MD5 hash
7. Success: redirect with toast

### Search (Index page)
- Input above table, `max-w-sm`
- Filters `personnel` array client-side by name or custom_id (case-insensitive contains)
- Instant results (no debounce needed for ~200 records)
- No results: table body shows empty, no special empty state
