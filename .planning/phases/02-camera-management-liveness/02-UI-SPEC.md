## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tight label spacing |
| sm | 8px | Compact element spacing, table cell vertical padding |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, page content margin |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Map container height on form page uses 256px (h-64). Map container on detail page uses 192px (h-48). These are fixed component heights, not spacing tokens.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.43 (20px) | Table cells, form descriptions, metadata text |
| Label | 14px (text-sm) | 500 (medium) | 1.43 (20px) | Form labels, table headers, sidebar nav items |

## Color

| Role | Token | Light Value | Dark Value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) | hsl(0 0% 3.9%) | Page background, content area |
| Secondary (30%) | `--card` / `--muted` | hsl(0 0% 100%) / hsl(0 0% 96.1%) | hsl(0 0% 3.9%) / hsl(0 0% 16.08%) | Cards, table header row, detail info section |
| Accent (10%) | `--primary` | hsl(0 0% 9%) | hsl(0 0% 98%) | Primary CTA button ("Add Camera", "Save"), active breadcrumb |
| Destructive | `--destructive` | hsl(0 84.2% 60.2%) | hsl(0 84% 60%) | Delete button, delete confirmation dialog button |

Accent reserved for:
- "Add Camera" primary button on the list page
- "Save" / "Update" submit button on create/edit forms
- Active sidebar nav item for Cameras

### Status Colors (Phase-Specific)

These are semantic colors used exclusively for camera online/offline indicators. They are NOT design system tokens -- they are applied via Tailwind utility classes.

| Status | Dot Color | Badge Style | Tailwind Classes |
|--------|-----------|-------------|------------------|
| Online | Green | Filled green dot + "Online" text | `bg-emerald-500` (dot), `text-emerald-700 dark:text-emerald-400` (text) |
| Offline | Gray | Filled gray dot + "Offline" text | `bg-neutral-400 dark:bg-neutral-500` (dot), `text-muted-foreground` (text) |

The status indicator is a custom composite element: a `6px` filled circle (`size-1.5 rounded-full`) followed by the status label. This is NOT the shadcn Badge component -- it is a simple inline flex layout for minimal visual weight in a table row context.

## Page Specifications

### Page 1: Camera List (`/cameras`)

**Route:** `cameras.index`
**Layout:** AppLayout (automatic via `app.ts` resolver -- no prefix match needed, falls through to default)
**Breadcrumbs:** `[{ title: 'Cameras', href: cameras.index() }]`

**Structure:**
```
+----------------------------------------------------------+
| Heading: "Cameras"                 [+ Add Camera] Button  |
+----------------------------------------------------------+
| <table>                                                   |
|   Name | Device ID | Location | Status    | Last Seen    |
|   -----|-----------|----------|-----------|-------------- |
|   ...  | ...       | ...      | * Online  | 2 min ago    |
|   ...  | ...       | ...      | * Offline | 15 min ago   |
+----------------------------------------------------------+
```

**Header area:**
- Page heading "Cameras" using `Heading.vue` with `variant="default"` (text-xl font-semibold)
- "Add Camera" `Button` (variant="default", size="default") positioned to the right of the heading in a flex container: `flex items-center justify-between`
- Button icon: `Plus` from lucide-vue-next, placed before text

**Table:**
- HTML `<table>` with classes: `w-full text-sm`
- Table header: `<thead>` row with `text-left text-muted-foreground font-medium` on `<th>` elements
- Table header cells: `px-4 py-3`
- Table body rows: `border-b border-border` on each `<tr>`, `hover:bg-muted/50` for hover state
- Table body cells: `px-4 py-3`
- Columns:
  1. **Name** -- clickable link (`<Link>`) to camera show page, `font-medium text-foreground hover:underline`
  2. **Device ID** -- `text-muted-foreground font-mono text-xs` (monospace for readability)
  3. **Location** -- plain text, truncated with `truncate max-w-[200px]`
  4. **Status** -- `CameraStatusDot` component (green/gray dot + label)
  5. **Last Seen** -- relative time ("2 min ago", "15 min ago"), `text-muted-foreground`. Display "Never" if `last_seen_at` is null.

**Empty state (no cameras registered):**
- Centered within the table area
- Card with padding `p-12`, centered content
- Icon: `Camera` from lucide-vue-next at `size-12 text-muted-foreground/50`
- Heading: "No cameras registered" at `text-lg font-medium`
- Body: "Add your first camera to start monitoring." at `text-sm text-muted-foreground`
- Button: "Add Camera" (variant="default") below the body text, `mt-4`

**Real-time behavior:**
- `useEcho('fras.alerts', '.CameraStatusChanged', callback)` updates the matching camera row's `is_online` and `last_seen_at` in local reactive state without page reload
- Status dot and last-seen text update reactively

**Interactions:**
- Clicking camera name navigates to `/cameras/{id}` (camera show page)
- Clicking "Add Camera" navigates to `/cameras/create`
- No bulk actions, no sorting, no filtering (D-11: max 8 cameras)

### Page 2: Camera Create (`/cameras/create`)

**Route:** `cameras.create`
**Layout:** AppLayout
**Breadcrumbs:** `[{ title: 'Cameras', href: cameras.index() }, { title: 'Add Camera', href: cameras.create() }]`

**Structure:**
```
+----------------------------------------------------------+
| Heading: "Add Camera"                                     |
| Description: "Register a new camera for monitoring"       |
+----------------------------------------------------------+
| <Form>                                                    |
|   [Device ID    ] [Name           ]                       |
|   [Location Label                  ]                       |
|   [Latitude      ] [Longitude      ]                       |
|   +------------------------------------+                   |
|   |         Mapbox Map (256px)          |                   |
|   |    Click to set camera location     |                   |
|   +------------------------------------+                   |
|   [Save]                                                   |
+----------------------------------------------------------+
```

**Heading area:**
- `Heading.vue` with `variant="default"`, title "Add Camera", description "Register a new camera for monitoring"

**Form:**
- Inertia `<Form>` with `v-bind="CameraController.store.form()"` and `v-slot="{ errors, processing }"`
- Max width: `max-w-2xl` (672px) to keep form comfortable and not stretch full-width
- Vertical spacing between form groups: `space-y-6`

**Form fields:**
1. **Device ID + Name** -- two-column grid: `grid grid-cols-2 gap-4`
   - Device ID: `<Input>` with placeholder "e.g. 1026700", required
   - Name: `<Input>` with placeholder "e.g. Main Entrance", required
2. **Location Label** -- full width
   - `<Input>` with placeholder "e.g. Building A, Ground Floor", required
3. **Latitude + Longitude** -- two-column grid: `grid grid-cols-2 gap-4`
   - Latitude: `<Input>` type="text" (not number, to preserve decimal string precision per Pitfall 6), placeholder "e.g. 8.9475785", required
   - Longitude: `<Input>` type="text", placeholder "e.g. 125.5406434", required
   - Help text below both fields: `text-xs text-muted-foreground` -- "Click the map below or type coordinates manually. Coordinates sync both ways."
4. **Mapbox Map** -- `MapboxMap.vue` with `interactive=true`, height `h-64` (256px), full width, rounded corners `rounded-lg overflow-hidden border border-border`
   - Map click sets lat/lng fields via `update:coordinates` emit
   - Typing in lat/lng fields updates map center and marker position via watcher
   - Default center: Butuan City coordinates (8.9475, 125.5406) if no coordinates entered
   - Zoom level: 15 (street level)
   - Style: Use user's appearance preference (dark/light) to select HelderDene's custom Mapbox styles

**Validation errors:**
- `<InputError>` component below each field group, using `errors.field_name` from Inertia form slot

**Submit:**
- `Button` (variant="default") with text "Save", disabled while `processing`
- On success: redirects to `cameras.index` with toast "Camera added."

### Page 4: Camera Detail (`/cameras/{camera}`)

**Route:** `cameras.show`
**Layout:** AppLayout
**Breadcrumbs:** `[{ title: 'Cameras', href: cameras.index() }, { title: camera.name, href: cameras.show(camera) }]`

**Structure:**
```
+----------------------------+----------------------------+
| Camera Information (Card)  | Enrolled Personnel (Card)  |
|                            |                            |
| Name: Main Entrance       | (empty state placeholder)  |
| Device ID: 1026700        |                            |
| Location: Building A, GF  |                            |
| Status: * Online           |                            |
| Last Seen: 2 min ago      |                            |
|                            |                            |
| +------------------------+ |                            |
| |   Mapbox Map (192px)   | |                            |
| +------------------------+ |                            |
|                            |                            |
| [Edit]  [Delete]           |                            |
+----------------------------+----------------------------+
```

**Layout:** Two-column grid: `grid grid-cols-1 lg:grid-cols-5 gap-6`
- Left column (camera info): `lg:col-span-3`
- Right column (enrolled personnel): `lg:col-span-2`

**Camera Information Card (left):**
- `Card` > `CardHeader` > `CardTitle` "Camera Information"
- `CardContent` with vertical layout `space-y-4`
- Each field displayed as definition-list style:
  - Label: `text-sm font-medium text-muted-foreground`
  - Value: `text-sm text-foreground` (below label, not inline)
- Fields displayed: Name, Device ID (monospace `font-mono`), Location, Status (`CameraStatusDot`), Last Seen (relative time or "Never")
- `Separator` after the fields
- Mapbox map: `MapboxMap.vue` with `interactive=false`, height `h-48` (192px), rounded corners, shows camera pin
- `Separator` after the map
- Action buttons in a flex row: `flex items-center gap-3`
  - "Edit" `Button` (variant="outline") linking to `cameras.edit(camera)`
  - "Delete" `Button` (variant="destructive", size="default") triggers delete dialog

**Enrolled Personnel Card (right):**
- `Card` > `CardHeader` > `CardTitle` "Enrolled Personnel"
- `CardContent` with empty state (D-16):
  - Icon: `Users` from lucide-vue-next at `size-10 text-muted-foreground/40`
  - Heading: "No personnel enrolled" at `text-sm font-medium`
  - Body: "Personnel will appear here once enrollment is configured in a future update." at `text-xs text-muted-foreground`
  - All centered: `flex flex-col items-center justify-center py-12 text-center`

**Delete Confirmation Dialog:**
- Triggered by the "Delete" button
- `Dialog` > `DialogContent`
- `DialogHeader`:
  - `DialogTitle`: "Delete camera?"
  - `DialogDescription`: "This will permanently remove {camera.name} and all associated enrollment records. This action cannot be undone."
- `DialogFooter` with `gap-2`:
  - "Cancel" `Button` (variant="secondary") via `DialogClose`
  - "Delete camera" `Button` (variant="destructive", disabled while processing)
- On confirm: `Form` with `v-bind="CameraController.destroy.form(camera)"`, redirects to `cameras.index` with toast "Camera deleted."

**Real-time behavior:**
- `useEcho` listener for `CameraStatusChanged` events updates status dot and last-seen on the detail page without reload

## Interaction States

### MapboxMap Component

| State | Behavior |
|-------|----------|
| Loading | Skeleton placeholder (`h-64` or `h-48`) with pulse animation until Mapbox tiles load |
| Interactive idle | Map displays with marker at current coordinates. Cursor shows crosshair on hover. |
| Interactive click | Marker moves to clicked position. `update:coordinates` emits new lat/lng. Coordinate input fields update. |
| Coordinate input change | Map pans to new center, marker repositions. Debounce coordinate input by 300ms. |
| Read-only | Map displays with marker. No click handler. Zoom/pan disabled. No crosshair cursor. |
| No coordinates (create page) | Map centered on Butuan City (8.9475, 125.5406) with zoom 15. No marker shown until first click or coordinate entry. |
| Error (token invalid) | Gray container with text "Map unavailable" centered in `text-sm text-muted-foreground` |

### Form Submissions

| State | Behavior |
|-------|----------|
| Idle | Submit button enabled, normal styling |
| Processing | Submit button disabled with `opacity-50`, cursor changes to `not-allowed` |
| Validation error | Red `InputError` text appears below invalid field, field border changes via `aria-invalid` |
| Success | Redirect to target page + toast notification |

### Delete Confirmation

| State | Behavior |
|-------|----------|
| Dialog closed | Delete button shows normally |
| Dialog open | Modal overlay, focus trapped inside dialog |
| Confirm processing | "Delete camera" button disabled, "Cancel" button still functional |
| Success | Dialog closes, redirect to camera list + toast "Camera deleted." |

### Real-Time Status Updates

| State | Behavior |
|-------|----------|
| Camera comes online | Status dot transitions to green, label changes to "Online", last-seen updates. No page reload. |
| Camera goes offline | Status dot transitions to gray, label changes to "Offline", last-seen remains at last known time. No page reload. |
| WebSocket disconnected | No visual indicator in this phase. (Status bar is Phase 6 scope, DASH-05.) |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| < 640px (mobile) | Table scrolls horizontally (`overflow-x-auto`). Form fields stack to single column. Detail page columns stack vertically. |
| 640-1023px (tablet) | Table fits naturally. Form fields remain two-column. Detail page columns stack vertically. |
| >= 1024px (desktop) | Full layout. Two-column form grids. Detail page uses 3/2 column split. |

Camera pages are admin-only and primarily used on desktop at the command center, but basic mobile responsiveness is maintained for field registration scenarios.

## Accessibility

| Element | Requirement |
|---------|-------------|
| Table | Use semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` elements |
| Status dot | `CameraStatusDot` includes `aria-label="Camera is online"` or `"Camera is offline"` |
| Map | `role="application"` on map container, `aria-label="Camera location map"` |
| Delete dialog | Focus trapped inside dialog (handled by reka-ui Dialog). Focus moves to "Cancel" button on open. |
| Form errors | `aria-invalid="true"` on invalid inputs (handled by shadcn Input component) |
| Camera name links | Descriptive link text (camera name itself is the link text -- no "click here") |
